# PO・TOROポータル 更新チェックリスト

ポータルを更新するときは、公開前にこの順番で確認してください。

## Googleスプレッドシート

- `assets/js/config.js` のCSV URLは、Googleスプレッドシートの「ウェブに公開」から発行された `output=csv` のURLにします。
- お知らせシートは、日付列と本文列を空にしないでください。
- イベントシートは、タイトル・日付・画像・タグ・リンクを1行ずつ入力します。
- グッズシートは、配布開始日・イベント名・グッズ名・配布終了日・備考を1行ずつ入力します。
- 週間予定シートは、`date`、`hours`、`note` の列名を使い、日付は `YYYY/MM/DD` または `YYYY-MM-DD` にします。
- 外部リンクを入れる場合は `https://` から始めます。

## 画像

- イベント画像は `img/event_YYYYMM_内容.png` または `img/event_YYYYMM_内容.jpg` のように、半角英数字中心の名前にします。
- イベント画像を追加したら、同じ名前に `.webp` を付けた画像も一緒に置きます。
- メイド服画像は `img/uniform_数字th.jpg` と `img/uniform_数字th.webp` をセットで置きます。
- 画像を差し替えたら、該当ページで拡大表示まで確認します。

## 公開前チェック

- `node scripts/validate-site.js` を実行し、エラーがないことを確認します。
- TOP、お知らせ、イベント、グッズ、週間お給仕表、料金、歴代メイド服、求人を一通り開きます。
- スマートフォン幅で、文字の重なりや横スクロールが出ていないか確認します。
- Google Sheetsが一時的に読めない場合でも、各ページに案内カードが出ることを確認します。
- GitHub Pagesの公開後、`https://potoro1001.github.io/potoro-portal/` を開いて反映を確認します。
